# True A2A Protocol Implementation Guide

## What's Implemented (Classifier Service)

### Architecture
```
classifier/
├── Program.cs                          # Entry point, A2A setup
├── classifier.csproj                  # Project file with FastEndpoints.A2A
├── Dockerfile
├── Infrastructure/
│   └── JwtService.cs                  # JWT token generation & validation
├── Services/
│   └── ServiceSettings.cs             # Configuration from environment
└── Skills/
    └── ClassifySkillEndpoint.cs       # A2A skill endpoint
```

### Key Points
1. **NO custom `/skills/*` endpoints** - Only `/a2a` JSON-RPC
2. **A2A Protocol Compliance**:
   - `/.well-known/agent-card.json` - Auto-generated by FastEndpoints.A2A
   - `/a2a` - JSON-RPC 2.0 endpoint for skill invocation
   - `SendMessage` - JSON-RPC method for calling skills

3. **Skills Definition**:
   - Registered with `.A2ASkill()` fluent API
   - Metadata: id, name, description, tags, examples
   - Input/output schemas auto-generated from DTOs

4. **Security**:
   - Bearer token validation on `/a2a` endpoint
   - JWT secret validation at startup (fail-fast if <32 chars)
   - Agent claims in JWT (agent_id, type)

## How to Call (via JSON-RPC)

```bash
curl -X POST http://classifier:5052/a2a \
  -H "Authorization: Bearer {JWT_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "1",
    "method": "classify-text",
    "params": {
      "text": "System is down"
    }
  }'
```

Response:
```json
{
  "jsonrpc": "2.0",
  "id": "1",
  "result": {
    "classification": "incident",
    "confidence": 0.95,
    "description": "Text classified as 'incident' with 95% confidence"
  }
}
```

## Remaining Services to Refactor

Same pattern for:
- **assessor**: Assess priority from classification
- **router**: Route to handler based on priority
- **handler**: Create ticket with full context
- **discovery**: Service registry (already partial)
- **identity**: Agent token issuer
- **api-backend**: Orchestrates via JSON-RPC calls

## API Backend Refactoring Needed

Current approach (custom HTTP POST to /skills/*):
```csharp
await client.PostAsJsonAsync("/skills/classify", request);
```

New approach (JSON-RPC to /a2a):
```csharp
var rpcRequest = new JsonRpcRequest
{
    JsonRpc = "2.0",
    Id = "1",
    Method = "classify-text",
    Params = new { text = input }
};
await client.PostAsJsonAsync("/a2a", rpcRequest);
```

## Testing

### 1. Health Check
```bash
curl http://localhost:5052/health
```

### 2. Agent Card Discovery
```bash
curl http://localhost:5052/.well-known/agent-card.json
```

### 3. Get Agent Token
```bash
curl "http://identity:5050/auth/agent/token?agentId=classifier-agent"
```

### 4. Call Skill via A2A
```bash
TOKEN=$(curl -s "http://identity:5050/auth/agent/token?agentId=agent1" | jq -r '.token')
curl -X POST http://classifier:5052/a2a \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "1",
    "method": "classify-text",
    "params": {"text": "Critical system down"}
  }'
```

## Notes

- FastEndpoints.A2A v1.0.0-beta.1 is used (only available version)
- Each service can be refactored independently
- Skills are only callable via JSON-RPC, NOT direct HTTP
- Agent cards are generated automatically from skill definitions
- Full type safety with C# records for requests/responses
